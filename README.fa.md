## چطور از این دوره استفاده کنیم

۱. روزی **یک جلسه** پیش برو (یا با سرعت خودت). عجله نکن — هر مثال را خودت تایپ کن.
۲. هر جلسه در پوشه‌ی [`sessions/`](sessions/) به‌صورت یک فایل README جداگانه است. از بالا تا پایین بخوان.
۳. کد قابل‌اجرای هر جلسه در پوشه‌ی [`examples/`](examples/) است. اجرا کن، خرابش کن، درستش کن.
۴. هر جلسه با **تمرین** تمام می‌شود. قبل از رفتن به جلسه‌ی بعد آن‌ها را انجام بده — یادگیری همان‌جا تثبیت می‌شود.
۵. این مخزن (repo) را روی گیت‌هاب نگه دار. تا جلسه‌ی ۲۰ همین می‌شود نمونه‌کار (پورتفولیو) تو.

### دو دستوری که مدام استفاده می‌کنی

</div>

```bash
go run path/to/file.go      # کامپایل + اجرا در یک مرحله (برای یادگیری)
go build ./...              # کامپایل همه‌چیز در یک فایل اجرایی
go test ./...               # اجرای همه‌ی تست‌ها
go fmt ./...                # قالب‌بندی خودکار کد (همیشه انجامش بده)
```

## 🏁 پروژه‌ی نهایی: «TaskFlow» — یک REST API مدیریت وظایف

تا جلسه‌ی ۲۰ تو **TaskFlow** را خواهی ساخت، یک بک‌اند با سبک محصول واقعی:

- ✅ یک API به‌صورت RESTful و JSON با کتابخانه‌ی استاندارد Go + یک روتر سبک
- ✅ ماندگاری داده روی PostgreSQL (یا SQLite) با مهاجرت‌های (migration) درست
- ✅ ثبت‌نام و ورود کاربر با **احراز هویت JWT**
- ✅ میان‌افزارها (لاگ، احراز هویت، بازیابی از panic، CORS)
- ✅ معماری تمیز و لایه‌ای (هندلر ← سرویس ← مخزن)
- ✅ تست‌های واحد و یکپارچگی
- ✅ Docker‌ای‌شده و آماده‌ی استقرار
- ✅ یک README تمیز تا کارفرما با یک دستور اجرایش کند

این همان نوع پروژه‌ای است که برایت مصاحبه‌ی **توسعه‌دهنده‌ی بک‌اند Go** می‌آورد.
