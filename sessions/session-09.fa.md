# جلسه‌ی ۰۹ — اشاره‌گرها 📍

**هدف (۱ ساعت):** یک بار برای همیشه اشاره‌گرها را روشن کنی. یاد می‌گیری اشاره‌گر
*چیست*، دو عملگر `&` و `*`، چرا توابع برای تغییر آرگومان‌ها به اشاره‌گر نیاز دارند،
و کِی (و کِی نه) از آن‌ها استفاده کنی. اشاره‌گرها ترسناک به نظر می‌رسند؛ در Go ساده
و امن‌اند.

> **مرور جلسه‌ی ۰۸:** از گیرنده‌ی *اشاره‌گری* (`func (r *T)`) استفاده کردی تا متدها
> استراکت را تغییر دهند. این جلسه `*` و `&` پشت آن را توضیح می‌دهد.

---

## ۱. اشاره‌گر چیست؟ (۱۰ دقیقه)

هر مقداری که برنامه‌ات ذخیره می‌کند، در یک **آدرس** در حافظه زندگی می‌کند. یک
**اشاره‌گر** فقط مقداری است که آن آدرس را نگه می‌دارد — به جایی که مقدار دیگری
زندگی می‌کند «اشاره می‌کند».

تشبیه: متغیر یک خانه است، مقدارش چیزی است که داخلش هست، و اشاره‌گر *آدرس خیابانی*
نوشته‌شده روی کاغذ است. با آدرس می‌توانی خانه را پیدا و محتوایش را عوض کنی.

</div>

```go
x := 10        // یک int عادی
p := &x        // p اشاره‌گری به x است (آدرس x را نگه می‌دارد)

fmt.Println(x)   // 10   — مقدار
fmt.Println(p)   // 0xc0000140a8 — یک آدرس (مال تو فرق دارد)
fmt.Println(*p)  // 10   — اشاره‌گر را دنبال کن تا مقدار را بگیری
```

<div dir="rtl">

دو عملگر همه‌ی چیزی است که نیاز داری:

| عملگر | نام | چه می‌کند |
|-------|-----|-----------|
| `&x` | «آدرسِ» | یک **اشاره‌گر به** `x` می‌دهد |
| `*p` | «دی‌رفرنس» | اشاره‌گر را دنبال می‌کند تا **مقدارِ** اشاره‌شده را بگیرد/تنظیم کند |

**نوع** اشاره‌گر به یک `int` به‌صورت `*int` نوشته می‌شود. اشاره‌گر به یک `User` می‌شود `*User`.

---

## ۲. خواندن و نوشتن از طریق اشاره‌گر (۱۰ دقیقه)

دی‌رفرنس با `*p` هر دو طرف کار می‌کند — می‌توانی هم بخوانی هم انتساب دهی:

</div>

```go
x := 10
p := &x

*p = 20          // مقدار آدرسی که p به آن اشاره می‌کند را عوض کن
fmt.Println(x)   // 20  ← x عوض شد، چون p به x اشاره می‌کند!
```

<div dir="rtl">

این کل هدف اشاره‌گرهاست: دو نام (`x` و `*p`) به **یک** قطعه حافظه اشاره می‌کنند، پس
تغییر یکی دیگری را عوض می‌کند.

### مقدار صفرِ اشاره‌گر `nil` است

اشاره‌گری که به جایی اشاره نمی‌کند `nil` است. دی‌رفرنس یک اشاره‌گر `nil` **panic**
می‌کند (کرش)، پس قبل از استفاده از اشاره‌گری که ممکن است nil باشد بررسی کن:

</div>

```go
var p *int        // nil — به جایی اشاره نمی‌کند
fmt.Println(p)    // <nil>
// fmt.Println(*p) // 💥 PANIC: دی‌رفرنس اشاره‌گر nil
```

## ۳. چرا اشاره‌گرها مهم‌اند: تغییر آرگومان تابع (۲۰ دقیقه)

این دلیل شماره‌ی ۱ نیاز مبتدی‌ها به اشاره‌گر است. **Go آرگومان‌ها را با مقدار پاس
می‌دهد — کپی‌شان می‌کند.** پس تابعی که یک `int` ساده می‌گیرد یک *کپی* می‌گیرد و
نمی‌تواند متغیر فراخواننده را عوض کند:

</div>

```go
func tryToDouble(n int) {
    n = n * 2          // فقط کپی را عوض می‌کند
}

x := 5
tryToDouble(x)
fmt.Println(x)   // هنوز 5 — اصلی دست نخورد
```

<div dir="rtl">

برای واقعاً تغییر دادن متغیر فراخواننده، یک **اشاره‌گر** به آن پاس بده:

</div>

```go
func double(n *int) {
    *n = *n * 2        // اشاره‌گر را دنبال کن، مقدار واقعی را عوض کن
}

x := 5
double(&x)             // آدرسِ x را پاس بده
fmt.Println(x)   // 10 — کار کرد!
```

<div dir="rtl">

> 🔑 **الگو:** پارامتر تابع `*T` است، با `&variable` صدایش می‌زنی، و درون آن از
> `*param` برای خواندن/نوشتن مقدار واقعی استفاده می‌کنی.

دقیقاً به همین خاطر گیرنده‌های *اشاره‌گری* متد (جلسه‌ی ۰۸) می‌توانند استراکت را عوض
کنند: گیرنده پنهانی یک اشاره‌گر به اصلی است.

فایل [`examples/session09/funcparams/funcparams.go`](../examples/session09/funcparams/funcparams.go) را اجرا کن.

## ۴. اشاره‌گر به استراکت — حالت رایج (۱۵ دقیقه)

در Go واقعی بیشتر اشاره‌گر را با **استراکت‌ها** استفاده می‌کنی، به دو دلیل: تغییرشان
در توابع، و دوری از کپی استراکت‌های بزرگ.

</div>

```go
type User struct {
    Name string
    Age  int
}

func birthday(u *User) {
    u.Age++          // Go می‌گذارد u.Age بنویسی نه (*u).Age — کوتاه‌نویسی خوب!
}

user := User{Name: "Sobhan", Age: 25}
birthday(&user)
fmt.Println(user.Age)   // 26
```

<div dir="rtl">

> 💡 **راحتی:** برای اشاره‌گرِ استراکت `u`، Go خودکار برای دسترسی به فیلد دی‌رفرنس
> می‌کند — `u.Age` یعنی `(*u).Age`. تقریباً هرگز خودت `(*u)` نمی‌نویسی.

### `new` و ساخت اشاره‌گر

</div>

```go
u1 := &User{Name: "Ali"}   // رایج‌ترین: آدرسِ یک لیترال استراکت
u2 := new(User)            // new(T) یک *T به مقدار صفر برمی‌گرداند
u2.Name = "Sara"
```

## ۵. کِی باید از اشاره‌گر استفاده کنی؟ (۵ دقیقه)

همه‌جا `*` نپاش. وقتی از اشاره‌گر استفاده کن که:

- ✅ یک تابع/متد باید آرگومان یا گیرنده‌اش را **تغییر** دهد.
- ✅ استراکت **بزرگ** است و کپی‌اش هدررفت است.
- ✅ باید «**بدون مقدار**» را با `nil` نشان دهی (مثلاً یک فیلد اختیاری).

وقتی مقدار ساده را ترجیح بده که:

- ✅ داده کوچک (یک `int`، استراکت کوچک) و فقط‌خواندنی است.
- ✅ امنیت و سادگی می‌خواهی — کپی مقداری را نمی‌توان غافلگیرانه تغییر داد.

> 🔑 **خبر خوب:** Go یک **جمع‌کننده‌ی زباله (garbage collector)** دارد، پس هرگز
> دستی حافظه آزاد نمی‌کنی. و حساب اشاره‌گری مثل C وجود ندارد — اشاره‌گرهای Go امن و ساده‌اند.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۱۰ انجام بده!)

فایل `examples/session09/practice/practice.go` را بساز:

۱. **swap:** `func swap(a, b *int)` بنویس که دو مقدار اشاره‌شده را جابه‌جا کند. تست: `x, y := 1, 2`، `swap(&x, &y)`، هر دو را چاپ کن.
۲. **increment:** `func increment(n *int)` بنویس که ۱ به مقدار اضافه کند. در حلقه ۵ بار روی یک متغیر صدایش بزن و نتیجه را چاپ کن (اگر از ۰ شروع شود باید ۵ شود).
۳. **تغییر استراکت:** `type Account struct { Balance float64 }`. `func deposit(a *Account, amount float64)` بنویس که موجودی را زیاد کند. اثبات کن حساب اصلی عوض شد.
۴. **مقدار در مقابل اشاره‌گر:** دو تابع — `func resetValue(a Account)` و `func resetPointer(a *Account)` — هر دو `Balance = 0`. نشان بده کدام واقعاً روی حساب فراخواننده اثر دارد و چرا (در کامنت).
۵. **بررسی nil:** `func describe(p *int)` بنویس که اگر `p` ناتهی است مقدار را چاپ کند وگرنه `"no value"`. با `&someInt` و یک `nil` صدایش بزن.
