# جلسه‌ی ۰۱ — سلام Go 🐹

**هدف (۱ ساعت):** بفهمی Go چیست، اولین برنامه‌ات را بنویسی و اجرا کنی، و ساختار
پایه‌ی هر فایل Go را یاد بگیری. در پایان، کد واقعی Go را روی دستگاه خودت اجرا می‌کنی.

---

## ۱. Go چیست؟ (۵ دقیقه)

Go (که گولنگ هم نامیده می‌شود) زبان برنامه‌نویسی‌ای است که در سال ۲۰۰۹ در گوگل
توسط رابرت گریزمر، راب پایک و کن تامپسون ساخته شد. این زبان برای این طراحی شده که:

- **ساده** باشد — زبانی کوچک که سریع یاد می‌گیری (فقط حدود ۲۵ کلیدواژه).
- **سریع** باشد — به یک فایل اجرایی بومی کامپایل می‌شود؛ نزدیک به سرعت C اجرا می‌شود.
- **برای همروندی** ساخته شده — انجام هم‌زمان چند کار آسان است (گوروتین‌ها).
- **برای بک‌اند عالی** است — API، میکروسرویس، ابزار خط فرمان و DevOps (داکر،
  کوبرنتیز و ترافورم همه با Go نوشته شده‌اند).

**چرا برای کار یادش بگیریم؟** تقاضا برای توسعه‌دهنده‌های Go در نقش‌های بک‌اند و
ابری بالاست، با این زبان سریع بهره‌ور می‌شوی، و درآمدش خوب است.

## ۲. ساختار یک برنامه‌ی Go (۱۰ دقیقه)

هر برنامه‌ی Go از **پکیج‌ها** ساخته می‌شود. این کوچک‌ترین برنامه‌ی کامل است:

</div>

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go!")
}
```

<div dir="rtl">

بیا **هر خط** را بشکافیم — این مهم است، با دقت بخوان:

| خط | معنی |
|----|------|
| `package main` | هر فایل Go به یک پکیج تعلق دارد. `main` ویژه است: نقطه‌ی ورود یک برنامه‌ی اجرایی. |
| `import "fmt"` | پکیج `fmt` («format») را از کتابخانه‌ی استاندارد می‌آورد — مسئول چاپ کردن است. |
| `func main()` | تابعی به نام `main`. وقتی برنامه را اجرا می‌کنی، Go اول این را صدا می‌زند. ورودی و خروجی ندارد. |
| `fmt.Println(...)` | تابع `Println` («چاپ خط») از پکیج `fmt`. متن را به‌علاوه‌ی یک خط جدید چاپ می‌کند. |
| `{ }` | آکولادها بدنه‌ی تابع را گروه‌بندی می‌کنند. در Go آکولاد باز `{` **باید** در همان خط باشد. |

> 🔑 **قاعده:** برنامه‌ای که بتوانی *اجرا* کنی باید `package main` و یک `func main()`
> داشته باشد. این دری است که Go از آن وارد می‌شود تا شروع کند.

## ۳. اولین برنامه‌ات را اجرا کن (۱۵ دقیقه)

فایل قابل‌اجرا در [`examples/session01/hello/hello.go`](../examples/session01/hello/hello.go)
است. یک ترمینال در پوشه‌ی دوره باز کن و اجرا کن:

</div>

```bash
go run examples/session01/hello/hello.go
```

<div dir="rtl">

باید این را ببینی:

</div>

```
Hello, Go!
```

<div dir="rtl">

🎉 همین حالا یک برنامه‌ی Go را کامپایل و اجرا کردی! دستور `go run` دو کار را
هم‌زمان انجام می‌دهد: کدت را به یک فایل اجرایی موقت **کامپایل** و سپس **اجرا** می‌کند.

### تفاوت `go run` و `go build`

</div>

```bash
go run examples/session01/hello/hello.go     # کامپایل + اجرا، فایلی باقی نمی‌ماند (عالی برای یادگیری)

go build -o hello examples/session01/hello/hello.go   # یک فایل اجرایی واقعی به نام hello می‌سازد
./hello                                                # آن فایل را مستقیم اجرا کن
```

<div dir="rtl">

جادوی Go: دستور `go build` **یک فایل کاملاً مستقل** بدون وابستگی می‌سازد. می‌توانی
آن یک فایل را به دستگاهی دیگر کپی کنی و فقط اجرا می‌شود.

## ۴. چاپ کردن: سه موردی که بیشترین استفاده را دارند (۱۰ دقیقه)

پکیج `fmt` سه تابع چاپ دارد که همیشه استفاده می‌کنی:

</div>

```go
fmt.Println("a", "b")        // چاپ: a b   (فاصله + خط جدید اضافه می‌کند)
fmt.Print("no newline")      // دقیقاً همان را چاپ می‌کند، بدون خط جدید
fmt.Printf("Name: %s\n", x)  // چاپ قالب‌بندی‌شده با «فعل‌ها» (verbs)
```

<div dir="rtl">

`Printf` از **فعل‌های قالب‌بندی** استفاده می‌کند — جای‌نگه‌دارهایی که با مقدار جایگزین می‌شوند:

| فعل | معنی | نمونه خروجی |
|-----|------|-------------|
| `%s` | رشته | `Sobhan` |
| `%d` | عدد صحیح (دهدهی) | `42` |
| `%f` | اعشاری | `3.140000` |
| `%t` | بولی | `true` |
| `%v` | **هر مقدار** (قالب پیش‌فرض) | برای همه‌چیز کار می‌کند |
| `%T` | **نوع** مقدار | `int`، `string` |
| `\n` | خط جدید | (شکست خط) |

وقتی شک داری از `%v` استفاده کن — هر مقداری را منطقی چاپ می‌کند. نمونه‌اش را در
[`examples/session01/printing/printing.go`](../examples/session01/printing/printing.go) ببین:

</div>

```bash
go run examples/session01/printing/printing.go
```

## ۵. کامنت‌ها و قالب‌بندی (۵ دقیقه)

</div>

```go
// این یک کامنت تک‌خطی است.

/*
   این یک کامنت
   چندخطی است.
*/
```

<div dir="rtl">

**همیشه کدت را قالب‌بندی کن** با ابزار رسمی. این همه‌ی بحث‌های سبک را تمام می‌کند:

</div>

```bash
go fmt ./...
```

<div dir="rtl">

این کار به‌صورت خودکار با تب تورفتگی می‌دهد، چیزها را تراز می‌کند و فاصله‌ها را
درست می‌کند. تیم‌های واقعی Go لازم می‌دانند همه‌ی کد `gofmt` شده باشد. همین حالا
این عادت را بساز.

## ۶. خطاهای رایج مبتدی‌ها (۵ دقیقه)

Go عمداً سخت‌گیر است — این «خطاها» به تو عادت‌های خوب یاد می‌دهند:

</div>

```go
import "fmt"   // ❌ اگر چیزی را import کنی و استفاده نکنی، خطا می‌دهد
```

<div dir="rtl">

> **«imported and not used»** — Go از کامپایل import بلااستفاده سر باز می‌زند. حذفش کن.

</div>

```go
x := 5         // ❌ اگر متغیری را تعریف کنی و هرگز استفاده نکنی، خطا می‌دهد
```

<div dir="rtl">

> **«declared and not used»** — همین برای متغیرهای محلی هم صدق می‌کند. یا استفاده‌اش کن یا حذفش کن.

اینها اول آزاردهنده‌اند اما کدبیس را تمیز نگه می‌دارند. بپذیرشان.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۲ انجام بده!)

یک فایل جدید `examples/session01/practice/practice.go` بساز و این‌ها را حل کن:

۱. **معرفی خودت:** سه خط چاپ کن — نامت، هدفت (مثلاً «می‌خواهم توسعه‌دهنده‌ی Go شوم») و تاریخ امروز — با `fmt.Println`.
۲. **استفاده از Printf:** جمله‌ی `"I am learning Go and it is awesome"` را با `Printf` و با `%s` برای کلمه‌های «Go» و «awesome» بساز.
۳. **انواع:** با `%T` نوع `42`، `3.14`، `"hi"` و `true` را چاپ کن. اول حدس بزن، بعد بررسی کن.
۴. **بیلد کن:** `go build -o myapp examples/session01/practice/practice.go` را اجرا کن، بعد `./myapp` را اجرا کن. ببین همان خروجی `go run` را می‌گیری.

> 💡 گیر کردی؟ دو مثال حل‌شده در این پوشه همه‌ی تکنیک‌های لازم را نشان می‌دهند.
