# جلسه‌ی ۰۵ — توابع پیشرفته 🛠️

**هدف (۱ ساعت):** از توابع پایه فراتر برو و ویژگی‌هایی را یاد بگیر که Go را *Go*
می‌کنند: **چند مقدار بازگشتی** (و الگوی `value, err` که همه‌جا هست)، توابع
**variadic**، **closureها** و **`defer`**. این‌ها را مسلط شو و کد واقعی Go را راحت
می‌خوانی.

> **مرور جلسه‌ی ۰۴:** با `for` حلقه می‌زنی و توابعی می‌نویسی/صدا می‌زنی که پارامتر
> می‌گیرند و مقدار برمی‌گردانند. حالا توابع را قوی‌تر می‌کنیم.

---

## ۱. چند مقدار بازگشتی (۱۵ دقیقه) — *امضای* Go

برخلاف اکثر زبان‌ها، یک تابع Go می‌تواند **بیش از یک مقدار** برگرداند. این ترفند
نادری نیست — قلب Go اصطلاحی است.

</div>

```go
func divide(a, b int) (int, int) {   // خارج‌قسمت و باقی‌مانده را برمی‌گرداند
    return a / b, a % b
}

q, r := divide(17, 5)   // q=3, r=2
```

<div dir="rtl">

### الگوی `value, err` (حفظش کن!)

مهم‌ترین قرارداد Go: توابعی که ممکن است شکست بخورند **دو** چیز برمی‌گردانند —
نتیجه *و* یک خطا. اگر خطا `nil` باشد، موفق شد.

</div>

```go
func sqrt(x float64) (float64, error) {
    if x < 0 {
        return 0, errors.New("cannot sqrt a negative number")
    }
    return math.Sqrt(x), nil   // خطای nil = موفقیت
}

result, err := sqrt(16)
if err != nil {
    fmt.Println("error:", err)
    return
}
fmt.Println("result:", result)
```

<div dir="rtl">

> 🔑 **در طول حرفه‌ات هزاران بار `if err != nil { ... }` می‌نویسی.** Go برای خطاهای
> عادی استثنا (exception) ندارد — به‌جایش، خطاها *مقادیری* هستند که برمی‌گردانی و
> بررسی می‌کنی. در جلسه‌ی ۱۱ عمیق می‌شویم؛ اینجا فقط شکلش را یاد بگیر.

وقتی فقط به بعضی خروجی‌ها اهمیت می‌دهی، بقیه را با `_` دور بینداز:

</div>

```go
_, remainder := divide(17, 5)   // خارج‌قسمت را نادیده بگیر
```

<div dir="rtl">

### مقادیر بازگشتی نام‌دار

می‌توانی خروجی‌ها را نام‌گذاری کنی. مثل متغیرهای از پیش تعریف‌شده عمل می‌کنند و یک
`return` خالی مقادیر فعلی‌شان را برمی‌گرداند:

</div>

```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return   // «return لخت» — x و y را می‌فرستد
}
```

<div dir="rtl">

از خروجی نام‌دار کم استفاده کن — برای توابع کوتاه خوب است اما در توابع طولانی به
خوانایی آسیب می‌زند.

فایل [`examples/session05/multireturn/multireturn.go`](../examples/session05/multireturn/multireturn.go) را اجرا کن.

## ۲. توابع variadic (۱۰ دقیقه) — پذیرش هر تعداد آرگومان

یک `...` قبل از نوع یعنی «صفر یا بیشتر از این‌ها». درون تابع، پارامتر یک **اسلایس** است.

</div>

```go
func sum(numbers ...int) int {
    total := 0
    for _, n := range numbers {
        total += n
    }
    return total
}

sum(1, 2, 3)        // 6
sum(10, 20, 30, 40) // 100
sum()               // 0
```

<div dir="rtl">

خودِ `fmt.Println` variadic است — به همین خاطر هر تعداد آرگومان می‌گیرد.

برای پاس دادن یک اسلایس موجود به تابع variadic، آن را با `...` «پخش» کن:

</div>

```go
nums := []int{1, 2, 3}
sum(nums...)   // اسلایس را به آرگومان‌های جدا پخش می‌کند
```

<div dir="rtl">

فایل [`examples/session05/variadic/variadic.go`](../examples/session05/variadic/variadic.go) را اجرا کن.

## ۳. توابع مقدار هستند؛ closureها (۱۵ دقیقه)

در Go توابع **مقدار درجه‌یک** هستند — می‌توانی در متغیر ذخیره‌شان کنی، به‌عنوان
آرگومان پاس بدهی و از توابع دیگر برگردانی.

</div>

```go
add := func(a, b int) int {   // تابع بی‌نام ذخیره‌شده در متغیر
    return a + b
}
fmt.Println(add(2, 3))   // 5
```

<div dir="rtl">

یک **closure** تابعی است که متغیرهای دامنه‌ی محل ساختش را «به یاد می‌سپارد». این قدرتمند است:

</div>

```go
func counter() func() int {
    count := 0
    return func() int {   // این تابع داخلی count را به یاد می‌سپارد
        count++
        return count
    }
}

next := counter()
fmt.Println(next()) // 1
fmt.Println(next()) // 2
fmt.Println(next()) // 3
```

<div dir="rtl">

هر فراخوانی `counter()` یک `count` تازه و مستقل می‌سازد. closureها روشی هستند که
Go بدون کلاس رفتار باحالت (stateful) دارد. در طول حرفه‌ات در میان‌افزار، هندلر و
callback می‌بینی‌شان.

فایل [`examples/session05/closures/closures.go`](../examples/session05/closures/closures.go) را اجرا کن.

## ۴. `defer` — پاک‌سازی‌ای که همیشه اجرا می‌شود (۱۵ دقیقه)

`defer` یک فراخوانی تابع را برای اجرا **هنگام بازگشت تابع دربرگیرنده** زمان‌بندی
می‌کند — مهم نیست چطور برگردد (بازگشت عادی یا حتی panic). ابزار Go برای پاک‌سازی است.

</div>

```go
func readFile() {
    file := open("data.txt")
    defer file.Close()   // تضمین می‌شود وقتی readFile برمی‌گردد اجرا شود

    // ... کار با فایل ...
    // حتی اگر زود برگردیم یا panic کنیم، file.Close() باز هم اجرا می‌شود
}
```

<div dir="rtl">

> 🔑 **چرا مهم است:** پاک‌سازی را *درست کنار* چیزی که نیاز به پاک‌سازی دارد
> می‌نویسی، پس نمی‌توانی فراموشش کنی.

**دو قاعده برای یادآوری:**

۱. **ترتیب LIFO:** چند `defer` به‌صورت معکوس اجرا می‌شوند (آخرین ورودی، اولین خروجی).
   ```go
   defer fmt.Println("1")
   defer fmt.Println("2")
   defer fmt.Println("3")
   // چاپ: 3, 2, 1
   ```
۲. **آرگومان‌ها فوراً ارزیابی می‌شوند**، اما فراخوانی در انتها اتفاق می‌افتد:
   ```go
   x := 10
   defer fmt.Println("deferred x:", x) // همین حالا 10 را می‌گیرد
   x = 20
   // هنگام بازگشت چاپ می‌کند "deferred x: 10"
   ```

فایل [`examples/session05/defer/defer.go`](../examples/session05/defer/defer.go) را اجرا کن.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۶ انجام بده!)

فایل `examples/session05/practice/practice.go` را بساز:

۵. **ترتیب defer:** تابعی با سه `defer fmt.Println(...)` بنویس و قبل از اجرا ترتیب خروجی را حدس بزن.
