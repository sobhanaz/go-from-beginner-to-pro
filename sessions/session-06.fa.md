# جلسه‌ی ۰۶ — آرایه‌ها و اسلایس‌ها 📚

**هدف (۱ ساعت):** یاد بگیری Go چطور فهرستی از مقادیر را نگه می‌دارد. با
**آرایه‌های** ثابت‌اندازه (که مستقیم کم استفاده می‌شوند) و **اسلایس‌ها** آشنا می‌شوی
— فهرست پویا و قابل‌رشد که پراستفاده‌ترین ساختمان داده در Go است. در پایان
`append`، برش، پیمایش، و دوری از تله‌ی کلاسیک اسلایس را بلدی.

> **مرور جلسه‌ی ۰۵:** توابع را مسلط شدی. حالا *مجموعه‌ای* از داده را نگه می‌داریم.

---

## ۱. آرایه‌ها — اندازه‌ی ثابت (۱۰ دقیقه)

یک **آرایه** تعداد ثابتی از عناصر هم‌نوع را نگه می‌دارد. اندازه بخشی از نوع است و
**نمی‌تواند تغییر کند**.

</div>

```go
var nums [3]int           // [0 0 0] — مقدار صفر
nums[0] = 10              // مقداردهی با اندیس (از صفر)
nums[1] = 20
fmt.Println(nums)         // [10 20 0]
fmt.Println(len(nums))    // 3

primes := [5]int{2, 3, 5, 7, 11}
days := [...]string{"Mon", "Tue", "Wed"} // طول 3 — Go می‌شمارد
```

<div dir="rtl">

**نکات کلیدی:**
- اندیس از صفر شروع می‌شود؛ `len()` طول را می‌دهد.
- اندازه برای همیشه ثابت است. `[3]int` و `[4]int` *انواع متفاوتی‌اند*.
- آرایه‌ها هنگام انتساب یا پاس به تابع **کپی** می‌شوند (معنای مقداری).

> 💡 در عمل به‌ندرت آرایه را مستقیم تعریف می‌کنی — **اسلایس‌ها** آنی هستند که واقعاً
> استفاده می‌کنی. اما آرایه‌ها زیر هر اسلایس وجود دارند، پس درکشان کن.

---

## ۲. اسلایس‌ها — فهرست پویایی که همه‌جا استفاده می‌کنی (۲۰ دقیقه)

یک **اسلایس** نمای انعطاف‌پذیر و قابل‌رشدی روی یک آرایه است. اندازه‌ی ثابت ندارد و
با افزودن عناصر رشد می‌کند. این *همان* مجموعه‌ی Go است.

</div>

```go
// لیترال — نبودن اندازه در براکت = اسلایس است، نه آرایه.
fruits := []string{"apple", "banana"}

// اسلایس خالی با make: make([]T, length, capacity)
scores := make([]int, 0)      // خالی، آماده‌ی رشد
buffer := make([]int, 3)      // [0 0 0]، طول 3
```

<div dir="rtl">

### رشد اسلایس با `append`

`append` عناصر را اضافه می‌کند و اسلایس **جدیدی** برمی‌گرداند. همیشه نتیجه را
دوباره انتساب بده:

</div>

```go
nums := []int{1, 2, 3}
nums = append(nums, 4)         // [1 2 3 4]
nums = append(nums, 5, 6, 7)   // چندتا با هم
nums = append(nums, others...) // افزودن یک اسلایس دیگر (پخش با ...)
```

<div dir="rtl">

> 🔑 **همیشه `nums = append(nums, x)` بنویس.** `append` ممکن است آرایه‌ی زیرین
> جدیدی بسازد، پس اسلایس به‌روزشده را *برمی‌گرداند*. نادیده گرفتن مقدار بازگشتی
> یک باگ کلاسیک مبتدی‌هاست.

### طول در مقابل ظرفیت

اسلایس **طول** (چند عنصر دارد) و **ظرفیت** (چند عنصر قبل از رشد آرایه‌ی زیرین جا
می‌گیرد) دارد:

</div>

```go
s := make([]int, 2, 5)
fmt.Println(len(s), cap(s))   // 2 5
```

<div dir="rtl">

معمولاً ظرفیت را دستی مدیریت نمی‌کنی، اما دانستن وجودش توضیح می‌دهد چرا `append`
کارآمد است (به‌جای یکی‌یکی، تکه‌تکه رشد می‌کند).

فایل [`examples/session06/slices/slices.go`](../examples/session06/slices/slices.go) را اجرا کن.

## ۳. برش — گرفتن زیراسلایس (۱۵ دقیقه)

نحو `s[low:high]` اسلایسی از اندیس `low` تا **ولی نه شاملِ** `high` می‌دهد:

</div>

```go
nums := []int{10, 20, 30, 40, 50}
fmt.Println(nums[1:3])   // [20 30]   (اندیس‌های 1, 2)
fmt.Println(nums[:2])    // [10 20]   (از ابتدا)
fmt.Println(nums[3:])    // [40 50]   (تا انتها)
fmt.Println(nums[:])     // [10 20 30 40 50] (کل)
```

<div dir="rtl">

بازه‌ی نیم‌باز (`low` شامل، `high` غیرشامل) یعنی `nums[1:3]` دقیقاً `3 - 1 = 2`
عنصر دارد. این قرارداد محاسبه‌ی طول را آسان می‌کند.

### ⚠️ تله‌ی کلاسیک: اسلایس‌ها حافظه را به اشتراک می‌گذارند

یک زیراسلایس به **همان آرایه‌ی زیرینِ** اصلی اشاره می‌کند. تغییر یکی می‌تواند آن
یکی را هم تغییر دهد:

</div>

```go
original := []int{1, 2, 3, 4, 5}
part := original[1:3]   // [2 3]، حافظه با original مشترک است
part[0] = 99
fmt.Println(original)   // [1 99 3 4 5]  ← original هم عوض شد!
```

<div dir="rtl">

برای گرفتن یک کپی مستقل از `copy` استفاده کن:

</div>

```go
dst := make([]int, len(part))
copy(dst, part)         // dst حالا یک کپی واقعی و جداست
```

<div dir="rtl">

این *همه* را اول گیر می‌اندازد. یادت باشد: برش یک *نما* است، نه کپی.

فایل [`examples/session06/slicing/slicing.go`](../examples/session06/slicing/slicing.go) را اجرا کن.

## ۴. پیمایش با `for range` (۱۰ دقیقه)

راه طبیعی قدم زدن روی اسلایس:

</div>

```go
fruits := []string{"apple", "banana", "cherry"}

for i, fruit := range fruits {
    fmt.Printf("%d: %s\n", i, fruit)
}

for _, fruit := range fruits {   // فقط مقدار؟ اندیس را با _ دور بینداز
    fmt.Println(fruit)
}

for i := range fruits {          // فقط اندیس؟ متغیر دوم را حذف کن
    fmt.Println(i)
}
```

<div dir="rtl">

فایل [`examples/session06/arrays/arrays.go`](../examples/session06/arrays/arrays.go) را برای آرایه‌ها اجرا کن، بعد بقیه را.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۷ انجام بده!)

فایل `examples/session06/practice/practice.go` را بساز:

۱. **ساخت فهرست:** از یک `[]int` خالی شروع کن، در یک حلقه اعداد ۱ تا ۱۰ را `append` کن، بعد اسلایس و طولش را چاپ کن.
۲. **جمع و میانگین:** با `nums := []float64{4, 8, 15, 16, 23, 42}`، با `range` جمع و میانگین را حساب و چاپ کن.
۳. **فیلتر:** `func evens(nums []int) []int` بنویس که فقط اعداد زوج ورودی را برگرداند. تست کن.
۴. **معکوس:** `func reverse(s []int) []int` بنویس که اسلایس جدیدی با عناصر معکوس برگرداند.
۵. **اثبات تله:** اسلایسی بساز، زیراسلایس با `[1:3]` بگیر، زیراسلایس را تغییر بده و اصلی را چاپ کن تا اثر حافظه‌ی مشترک را *ببینی*. بعد با `copy` دوباره انجام بده و نشان بده اصلی دست‌نخورده می‌ماند.
