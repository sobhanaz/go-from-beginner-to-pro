# جلسه‌ی ۰۳ — کنترل جریان 🔀

**هدف (۱ ساعت):** به برنامه‌هایت یاد بده تصمیم بگیرند. `if`/`else`، `switch`
قدرتمند Go، عملگرهایی که آن‌ها را می‌رانند، و ترفند `iota` برای ثابت‌های تمیز را
مسلط می‌شوی.

> **مرور جلسه‌ی ۰۲:** متغیرها (`:=`، `var`)، چهار نوع پایه و مقدار صفرشان، و این
> که Go هرگز خودکار تبدیل نوع نمی‌کند.

---

## ۱. عملگرها — بلوک‌های سازنده‌ی تصمیم‌ها (۱۰ دقیقه)

تصمیم‌ها از عبارت‌هایی ساخته می‌شوند که به `true` یا `false` ارزیابی می‌شوند.

**عملگرهای مقایسه‌ای** (نتیجه همیشه `bool` است):

| عملگر | معنی | نمونه |
|-------|------|-------|
| `==` | مساوی | `age == 18` |
| `!=` | نامساوی | `name != ""` |
| `<` `>` | کوچک‌تر / بزرگ‌تر | `score > 50` |
| `<=` `>=` | کوچک‌تر/بزرگ‌تر-مساوی | `age >= 18` |

**عملگرهای منطقی** (ترکیب بولی‌ها):

| عملگر | معنی | درست وقتی... |
|-------|------|---------------|
| `&&` | و (AND) | **هر دو** طرف درست باشند |
| `\|\|` | یا (OR) | **حداقل یکی** درست باشد |
| `!` | نقیض (NOT) | true↔false را برعکس می‌کند |

**حساب** را قبلاً دیدی: `+ - * /` و `%` (باقی‌مانده). عملگر `%` فوق‌العاده مفید
است: `n % 2 == 0` زوج بودن `n` را تست می‌کند.

## ۲. `if` / `else if` / `else` (۱۵ دقیقه)

شکل پایه — **توجه: بدون پرانتز دور شرط**، اما آکولاد حتی برای یک خط **اجباری** است:

</div>

```go
score := 75

if score >= 90 {
    fmt.Println("A")
} else if score >= 70 {
    fmt.Println("B")
} else {
    fmt.Println("Needs improvement")
}
```

<div dir="rtl">

> 🔑 سبک Go: آکولاد باز `{` همیشه در همان خط `if` است. این اختیاری نیست — `gofmt`
> اجرایش می‌کند و کامپایلر انتظارش را دارد.

### `if` با یک عبارت کوتاه (بسیار اصطلاحی در Go!)

می‌توانی *قبل* از شرط یک عبارت کوچک اجرا کنی که با `;` جدا می‌شود. متغیری که
می‌سازد **فقط درون `if`/`else` دیده می‌شود** — این دامنه را تنگ نگه می‌دارد:

</div>

```go
if age := getAge(); age >= 18 {
    fmt.Println("Adult, age is", age)
} else {
    fmt.Println("Minor, age is", age)
}
// age اینجا وجود ندارد
```

<div dir="rtl">

این الگو را *همه‌جا* در Go واقعی می‌بینی، مخصوصاً با خطاها:

</div>

```go
if err := doSomething(); err != nil {
    // مدیریت خطا
}
```

## ۳. `switch` — تمیزتر از زنجیره‌های طولانی if/else (۲۰ دقیقه)

وقتی یک مقدار را با گزینه‌های زیاد مقایسه می‌کنی، `switch` واضح‌تر است:

</div>

```go
day := "Sat"

switch day {
case "Sat", "Sun":          // چند مقدار در هر case!
    fmt.Println("Weekend 🎉")
case "Mon":
    fmt.Println("Monday...")
default:
    fmt.Println("A weekday")
}
```

<div dir="rtl">

**دو چیز که کسانی که از زبان‌های دیگر می‌آیند را غافلگیر می‌کند:**

۱. **به `break` نیازی نیست.** Go خودکار بعد از case منطبق متوقف می‌شود — هیچ
   «افتادن» تصادفی به case بعدی وجود ندارد.
۲. **یک case می‌تواند چند مقدار** را با کاما لیست کند (`case "Sat", "Sun":`).

### `switch` بدون شرط (جایگزین زنجیره‌ی if/else)

مقدار را حذف کن و هر `case` یک تست بولی می‌شود. این تمیزترین راه برای بازه/آستانه است:

</div>

```go
switch {
case score >= 90:
    fmt.Println("A")
case score >= 70:
    fmt.Println("B")
default:
    fmt.Println("C")
}
```

<div dir="rtl">

> 💡 `fallthrough` وجود دارد اگر *واقعاً* بخواهی به case بعدی ادامه دهی، اما نادر
> است. در مصاحبه ذکرش کن؛ کم استفاده‌اش خواهی کرد.

## ۴. `iota` — شماره‌گذاری خودکار ثابت‌ها (۱۰ دقیقه)

وقتی به مجموعه‌ای از ثابت‌های مرتبط نیاز داری (مثل وضعیت‌ها یا سطوح)، تایپ دستی
`0, 1, 2, 3` خطاخیز است. `iota` درون یک بلوک `const` خودکار افزایش می‌یابد:

</div>

```go
const (
    Sunday = iota   // 0
    Monday          // 1
    Tuesday         // 2
    Wednesday       // 3
)
```

<div dir="rtl">

`iota` از `0` در اولین خط بلوک `const` شروع و در هر خط ۱ تا اضافه می‌شود. این راه
اصطلاحی ساخت **شمارش‌ها (enum)** در Go است.

یک الگوی حرفه‌ای رایج از یک نوع نام‌دار برای امنیت استفاده می‌کند:

</div>

```go
type Status int

const (
    StatusPending Status = iota   // 0
    StatusActive                  // 1
    StatusClosed                  // 2
)
```

<div dir="rtl">

حالا یک مقدار `Status` فقط می‌تواند یکی از آن سه باشد — کدت خودتوضیح است. این را
در پروژه‌ی نهایی برای چیزهایی مثل وضعیت وظیفه استفاده می‌کنی.

فایل [`examples/session03/iota/iota.go`](../examples/session03/iota/iota.go) را اجرا کن.
