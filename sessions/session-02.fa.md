# جلسه‌ی ۰۲ — متغیرها و انواع داده 🧮

**هدف (۱ ساعت):** یاد بگیری Go چطور داده را نگه می‌دارد. دو راه تعریف متغیر را
مسلط می‌شوی، انواع پایه را می‌فهمی، مفهوم بسیار مهم **مقدار صفر** را یاد می‌گیری،
با ثابت‌ها کار می‌کنی و بین انواع تبدیل می‌کنی.

> **مرور جلسه‌ی ۰۱:** هر برنامه‌ی اجرایی `package main` + `func main()` دارد، و با
> `fmt.Println`/`Printf` چاپ می‌کنیم. حالا مدام از آن‌ها استفاده می‌کنیم.

---

## ۱. متغیر چیست؟ (۵ دقیقه)

یک **متغیر** جعبه‌ای نام‌دار است که یک مقدار را نگه می‌دارد. در Go هر متغیر یک
**نوع** (رشته، عدد صحیح و...) دارد که بعد از تعریف هرگز عوض نمی‌شود. Go
**نوع‌گذاری ایستا** دارد — نوع در زمان کامپایل ثابت است، که دسته‌ی کاملی از باگ‌ها
را قبل از اجرا می‌گیرد.

---

## ۲. تعریف متغیرها — دو راه (۱۵ دقیقه)

### راه ۱: `var` (صریح)

</div>

```go
var name string = "Sobhan"
var age int = 25
```

<div dir="rtl">

می‌توانی بگذاری Go نوع را از روی مقدار **استنباط** کند (لازم نیست `string` بنویسی):

</div>

```go
var name = "Sobhan"   // Go یک رشته می‌بیند، پس name رشته است
var age = 25          // Go یک عدد کامل می‌بیند، پس age از نوع int است
```

<div dir="rtl">

همچنین می‌توانی *بدون* مقدار تعریف کنی — آن‌وقت **مقدار صفر** می‌گیرد (بخش بعد):

</div>

```go
var count int       // count برابر 0 است
var message string  // message برابر "" (رشته‌ی خالی) است
```

<div dir="rtl">

### راه ۲: `:=` (تعریف کوتاه) — همانی که بیشترین استفاده را می‌کنی

درون یک تابع، `:=` در یک مرحله تعریف **و** مقداردهی می‌کند، با استنباط نوع:

</div>

```go
name := "Sobhan"   // مثل: var name string = "Sobhan"
age := 25          // مثل: var age int = 25
```

<div dir="rtl">

> 🔑 **دو قاعده‌ی کلیدی برای `:=`**
> ۱. فقط **درون یک تابع** کار می‌کند (نه در سطح سراسری — آنجا از `var` استفاده کن).
> ۲. متغیر باید **جدید** باشد. برای فقط مقداردهی مجدد یک متغیر موجود از `=` استفاده کن:
> ```go
> age := 25   // تعریف
> age = 26    // مقداردهی مجدد (بدون کولن)
> ```

### تعریف چندتا با هم

</div>

```go
var (
    firstName = "Sobhan"
    lastName  = "Azimzadeh"
    age       = 25
)

// یا با :=
x, y := 10, 20
```

<div dir="rtl">

فایل [`examples/session02/variables/variables.go`](../examples/session02/variables/variables.go) را اجرا کن.

## ۳. انواع پایه (۱۵ دقیقه)

انواع داخلی Go که روزانه استفاده می‌کنی:

| دسته | انواع | نمونه |
|------|-------|-------|
| **رشته** | `string` | `"hello"` |
| **عدد صحیح** | `int`، `int8/16/32/64`، `uint`، ... | `42`، `-7` |
| **اعشاری** | `float32`، `float64` | `3.14`، `-0.5` |
| **بولی** | `bool` | `true`، `false` |
| **بایت/rune** | `byte` (=uint8)، `rune` (=int32) | در جلسه‌ی ۰۷ |

**توصیه‌ی عملی برای مبتدی‌ها — فقط از این پیش‌فرض‌ها استفاده کن:**
- عدد کامل ← `int`
- عدد اعشاری ← `float64`
- متن ← `string`
- درست/غلط ← `bool`

انواع اندازه‌دار (`int8`، `uint32` و...) وقتی مهم‌اند که به حافظه یا قالب‌های دودویی
اهمیت بدهی. خودت می‌فهمی کِی لازمشان داری. روی دستگاه‌های امروزی `int` معادل ۶۴ بیتی است.

فایل [`examples/session02/types/types.go`](../examples/session02/types/types.go) را ببین.

## ۴. مقدار صفر — Go هیچ «تعریف‌نشده‌ای» ندارد (۱۰ دقیقه)

این یکی از بهترین ویژگی‌های Go است. **هر متغیر همیشه مقداردهی اولیه می‌شود.**
اگر مقداری ندهی، Go **مقدار صفرِ** نوعش را می‌دهد:

| نوع | مقدار صفر |
|-----|-----------|
| `int`، `float64` | `0` |
| `string` | `""` (رشته‌ی خالی) |
| `bool` | `false` |
| اشاره‌گر، اسلایس، مپ و... | `nil` (جلسات بعد) |

> 🔑 **چرا مهم است:** در خیلی زبان‌ها متغیر مقداردهی‌نشده آشغال یا `undefined` دارد
> و باعث کرش می‌شود. در Go، متغیر تازه‌تعریف‌شده همیشه برای استفاده امن است.

---

## ۵. ثابت‌ها (۵ دقیقه)

یک **ثابت** مقداری است که هرگز عوض نمی‌شود. با `const` تعریف کن:

</div>

```go
const Pi = 3.14159
const AppName = "TaskFlow"
const MaxUsers = 100
```

<div dir="rtl">

از ثابت‌ها برای مقادیری استفاده کن که برای همیشه ثابت‌اند (نام تنظیمات، ثابت‌های
ریاضی، سقف‌های ثابت). تلاش برای مقداردهی مجدد یک ثابت خطای کامپایل است — دقیقاً همان چیزی که می‌خواهی.

---

## ۶. تبدیل نوع — Go هرگز برایت تبدیل نمی‌کند (۱۰ دقیقه)

Go سخت‌گیر است: انواع را به‌صورت خودکار قاطی **نمی‌کند**. این خطای کامپایل است:

</div>

```go
var x int = 5
var y float64 = 2.5
// z := x + y   // ❌ خطا: انواع ناهمخوان int و float64
```

<div dir="rtl">

باید با `T(value)` **صریحاً تبدیل** کنی:

</div>

```go
var x int = 5
var y float64 = 2.5
z := float64(x) + y   // ✅ اول x را به float64 تبدیل کن → 7.5
```

```go
a := 10
b := 3
fmt.Println(a / b)              // 3   (تقسیم صحیح — کوتاه می‌شود!)
fmt.Println(float64(a) / float64(b)) // 3.3333... (تقسیم اعشاری)
```

<div dir="rtl">

> ⚠️ **تله‌ی کلاسیک مبتدی‌ها:** `10 / 3` برابر `3` است، نه `3.33`، چون هر دو عدد
> صحیح‌اند و تقسیم صحیح باقی‌مانده را دور می‌اندازد. وقتی اعشار می‌خواهی به
> `float64` تبدیل کن.

تبدیل عدد ↔ رشته به پکیج `strconv` نیاز دارد (جلسه‌ی ۱۴)، **نه** یک تبدیل ساده —
`string(65)` مقدار `"A"` می‌دهد (یک کاراکتر)، نه `"65"`!

فایل [`examples/session02/conversion/conversion.go`](../examples/session02/conversion/conversion.go) را ببین.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۳ انجام بده!)

فایل `examples/session02/practice/practice.go` را بساز:

۱. **کارت پروفایل:** `name` (رشته)، `age` (int)، `height` (float64) و `student` (bool) را با **`:=`** تعریف کن. با `Printf` هرکدام را در یک خط با فعل درست چاپ کن.
۲. **مقدار صفر:** `var a int`، `var b string`، `var c bool` را **بدون مقدار** تعریف و چاپ کن. اول خروجی را حدس بزن، بعد بررسی کن.
۳. **تله‌ی تقسیم:** `7 / 2` و سپس `float64(7) / float64(2)` را چاپ کن. در یک کامنت برای خودت توضیح بده چرا فرق دارند.
۴. **تبدیل:** `var meters float64 = 1.81` داری. آن را به `int` تبدیل و هر دو را چاپ کن. با بخش اعشاری چه شد؟
۵. **ثابت‌ها:** یک `const TaxRate = 0.09` بساز. با `price := 100.0` قیمت نهایی با مالیات را حساب و چاپ کن.
