# جلسه‌ی ۰۷ — مپ‌ها، رشته‌ها و runeها 🗺️

**هدف (۱ ساعت):** انبار کلید/مقدار Go (**مپ**) را یاد بگیری، و بالاخره بفهمی یک
**رشته** واقعاً زیر پوسته چیست — از جمله تفاوت `byte` و `rune` و این که چرا لحظه‌ای
که به متن غیرانگلیسی یا ایموجی دست می‌زنی مهم می‌شود.

> **مرور جلسه‌ی ۰۶:** اسلایس‌ها فهرست‌های مرتب و قابل‌رشدند. مپ‌ها همتای *نامرتبِ*
> جست‌وجو-با-کلید هستند.

---

## ۱. مپ‌ها — جست‌وجوی کلید/مقدار (۲۰ دقیقه)

یک **مپ** **کلیدها** را به **مقادیر** ربط می‌دهد. مثل یک دیکشنری: یک کلمه (کلید)
را جست‌وجو کن تا تعریفش (مقدار) را بگیری. جست‌وجوها سریع‌اند (تقریباً O(1)).

</div>

```go
ages := map[string]int{
    "Alice": 30,
    "Bob":   25,
}

fmt.Println(ages["Alice"])   // 30
ages["Carol"] = 28           // افزودن یا به‌روزرسانی
ages["Alice"] = 31           // به‌روزرسانی موجود
delete(ages, "Bob")          // حذف یک کلید
fmt.Println(len(ages))       // تعداد ورودی‌ها
```

<div dir="rtl">

### ساخت مپ

</div>

```go
m1 := map[string]int{}          // لیترال خالی
m2 := make(map[string]int)      // همان، با make
var m3 map[string]int           // مپ nil — می‌توان خواند ولی نوشتن panic می‌کند!
```

<div dir="rtl">

> ⚠️ یک مپ `nil` (تعریف‌شده با `var` بدون مقدار) قابل خواندن است اما **نوشتن روی
> آن panic می‌کند**. همیشه قبل از افزودن کلید، مپ را با لیترال یا `make` بساز.

### اصطلاح «comma ok» — آیا کلید وجود دارد؟

خواندن کلیدِ نبود **مقدار صفر** برمی‌گرداند، پس نمی‌توانی «نبود» را از «هست ولی صفر»
تشخیص بدهی. راه‌حل فرم دومقداری است:

</div>

```go
age, ok := ages["Dave"]
if ok {
    fmt.Println("Dave is", age)
} else {
    fmt.Println("No Dave in the map")
}
```

<div dir="rtl">

`ok` فقط وقتی `true` است که کلید واقعاً وجود داشته باشد. این الگوی **comma-ok**
اصطلاحی Go است.

### پیمایش مپ

</div>

```go
for name, age := range ages {
    fmt.Printf("%s is %d\n", name, age)
}
```

<div dir="rtl">

> 🔑 **ترتیب مپ تصادفی است!** Go عمداً ترتیب پیمایش را تصادفی می‌کند تا هرگز اشتباهاً
> به آن وابسته نشوی. اگر خروجی مرتب می‌خواهی، کلیدها را در یک اسلایس جمع و مرتب کن
> (در جلسه‌ی ۱۴ از `sort` استفاده می‌کنیم).

فایل [`examples/session07/maps/maps.go`](../examples/session07/maps/maps.go) را اجرا کن.

## ۲. رشته‌ها — بایت‌های تغییرناپذیر (۱۵ دقیقه)

یک `string` در Go یک **دنباله‌ی فقط‌خواندنی از بایت‌ها** است. دو حقیقت مهم:

۱. **رشته‌ها تغییرناپذیرند** — نمی‌توانی یک کاراکتر را سرجایش عوض کنی. برای «تغییر»
   یک رشته، رشته‌ی جدیدی می‌سازی.
۲. **رشته بایت‌های کدگذاری‌شده‌ی UTF-8 است.** برای انگلیسی ساده، ۱ کاراکتر = ۱ بایت.
   برای دیگر خط‌ها (فارسی، چینی) و ایموجی، **یک کاراکتر می‌تواند چند بایت** باشد.

</div>

```go
s := "Hello"
fmt.Println(len(s))      // 5  (تعداد بایت‌ها)
fmt.Println(s[0])        // 72 (مقدار بایتِ 'H'، نه "H"!)
fmt.Printf("%c\n", s[0]) // H  (%c یک بایت/rune را به‌صورت کاراکتر قالب می‌دهد)
```

<div dir="rtl">

> ⚠️ `len(s)` **بایت‌ها** را می‌شمارد، نه کاراکترها. `len("héllo")` برابر ۶ است نه
> ۵، چون `é` در UTF-8 دو بایت است. همین برای `len("سلام")` — بایت است نه حروف.

### عملیات رایج رشته

پکیج `strings` (گشت کامل در جلسه‌ی ۱۴) کمک‌کننده‌های روزمره را دارد:

</div>

```go
strings.ToUpper("hello")            // "HELLO"
strings.Contains("hello", "ell")    // true
strings.Split("a,b,c", ",")         // ["a" "b" "c"]
strings.ReplaceAll("aaa", "a", "b") // "bbb"
strings.TrimSpace("  hi  ")         // "hi"
strings.Join([]string{"a", "b"}, "-") // "a-b"
```

## ۳. بایت در مقابل rune — مدیریت هر زبانی (۲۰ دقیقه)

این بخشی است که همه را گیج می‌کند، پس آرام برو.

- یک **`byte`** یک مقدار ۸ بیتی (`uint8`) است. رشته از بایت ساخته می‌شود.
- یک **`rune`** یک *کاراکترِ* یونیکد (`int32`) است. روش Go برای گفتن «یک کاراکتر
  منطقی، فارغ از این که چند بایت بگیرد».

وقتی به **کاراکترها** (نه بایت‌ها) نیاز داری، به `[]rune` تبدیل کن یا روی رشته
`range` بزن (که runeها را می‌دهد):

</div>

```go
word := "héllo"

for i, r := range word {   // range روی رشته runeها را با اندیس بایت می‌دهد
    fmt.Printf("byte %d: %c (rune %d)\n", i, r, r)
}

fmt.Println(len(word))            // 6  (بایت‌ها)
fmt.Println(len([]rune(word)))    // 5  (کاراکترها)
fmt.Println(utf8.RuneCountInString(word)) // 5
```

<div dir="rtl">

> 🔑 **قاعده‌ی سرانگشتی:**
> - با داده‌ی خام / ASCII کار می‌کنی؟ بایت‌ها خوب‌اند.
> - کاراکترهای واقعی (فارسی، ایموجی، حروف لهجه‌دار) را می‌شماری یا اندیس می‌گذاری؟
>   از **rune** استفاده کن (`[]rune(s)` یا `range`)، هرگز `s[i]`.

### ساخت کارآمد رشته

چون رشته‌ها تغییرناپذیرند، انجام مکرر `s = s + x` در حلقه هدررفت است (هر بار رشته‌ی
جدید می‌سازد). برای الحاق زیاد از `strings.Builder` استفاده کن:

</div>

```go
var b strings.Builder
for i := 0; i < 5; i++ {
    b.WriteString("go ")
}
result := b.String()   // "go go go go go "
```

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۸ انجام بده!)

فایل `examples/session07/practice/practice.go` را بساز:

۱. **شمارنده‌ی کلمه:** یک جمله را روی فاصله split کن و با `map[string]int` تعداد تکرار هر کلمه را بشمار. مپ را چاپ کن.
۲. **comma-ok:** یک `map[string]float64` از قیمت محصولات بساز. محصولی که هست و یکی که نیست را با اصطلاح comma-ok جست‌وجو و گزارش کن.
۳. **شمارنده‌ی واکه:** `func countVowels(s string) int` بنویس که واکه‌ها را با `range` (runeها)، بدون حساسیت به بزرگی/کوچکی، بشمارد.
۴. **معکوس رشته به‌درستی:** `func reverse(s string) string` بنویس که با `[]rune` (نه بایت) معکوس کند تا روی `"héllo"` و ایموجی کار کند. تست کن.
۵. **طول کاراکتر در مقابل بایت:** هم `len(s)` و هم `len([]rune(s))` را برای رشته‌ای با کاراکتر غیر-ASCII چاپ کن و تفاوت را در کامنت توضیح بده.
