# جلسه‌ی ۰۴ — حلقه‌ها و توابع 🔁

**هدف (۱ ساعت):** با حلقه‌ی `for` (تنها حلقه‌ی Go — همه‌کاره است) کار را تکرار کنی
و کد را در **توابع** قابل‌استفاده‌ی مجدد سازمان دهی. این دو ابزار ستون فقرات هر
برنامه‌ای هستند که می‌نویسی.

> **مرور جلسه‌ی ۰۳:** با `if`/`switch` شاخه می‌زنی و با `&&`، `||`، `!` شرط‌ها را
> ترکیب می‌کنی. حالا تکرار و قابلیت استفاده‌ی مجدد را اضافه می‌کنیم.

---

## ۱. حلقه‌ی `for` — تنها حلقه‌ی Go (۲۰ دقیقه)

خیلی زبان‌ها `for`، `while` و `do-while` دارند. **Go فقط `for` دارد** — و همه‌ی
حالت‌ها را پوشش می‌دهد. شکل‌هایش را یاد بگیر:

### شکل ۱: حلقه‌ی سه‌بخشی کلاسیک

</div>

```go
for i := 0; i < 5; i++ {
    fmt.Println(i)   // چاپ 0 1 2 3 4
}
```

<div dir="rtl">

سه بخش جداشده با `;`:
۱. **مقداردهی اولیه**: `i := 0` — یک بار در ابتدا اجرا می‌شود.
۲. **شرط**: `i < 5` — قبل از هر تکرار بررسی می‌شود؛ تا وقتی درست باشد ادامه می‌یابد.
۳. **پس‌رو (post)**: `i++` — بعد از هر تکرار اجرا می‌شود. (`i++` یعنی `i = i + 1`.)

### شکل ۲: حلقه‌ی «while» (فقط یک شرط)

مقداردهی و پس‌رو را حذف کن — حالا مثل `while` رفتار می‌کند:

</div>

```go
n := 1
for n < 100 {
    n = n * 2
}
fmt.Println(n)   // 128
```

<div dir="rtl">

### شکل ۳: حلقه‌ی بی‌نهایت

</div>

```go
for {
    // تا وقتی بیرون نزنی برای همیشه اجرا می‌شود
    break   // از حلقه خارج می‌شود
}
```

<div dir="rtl">

این را برای سرورها و حلقه‌های رویداد استفاده می‌کنی. از `break` برای خروج و
`continue` برای پرش به تکرار بعد استفاده کن:

</div>

```go
for i := 0; i < 10; i++ {
    if i == 3 {
        continue   // 3 را رد کن
    }
    if i == 6 {
        break      // در 6 کاملاً متوقف شو
    }
    fmt.Println(i) // 0 1 2 4 5
}
```

<div dir="rtl">

### شکل ۴: `for range` — پیمایش روی مجموعه‌ها

رایج‌ترین شکل در کد واقعی. روی عناصر یک اسلایس، رشته، مپ و... قدم می‌زند (از جلسه‌ی
۰۶ زیاد استفاده می‌کنیم):

</div>

```go
names := []string{"Ann", "Bob", "Cy"}
for index, name := range names {
    fmt.Println(index, name)   // 0 Ann / 1 Bob / 2 Cy
}
```

<div dir="rtl">

اگر به اندیس نیاز نداری از شناسه‌ی خالی `_` استفاده کن:

</div>

```go
for _, name := range names {
    fmt.Println(name)
}
```

<div dir="rtl">

> 🔑 **`_` (آندرلاین)** جای‌نگه‌دارِ «این مقدار را نمی‌خواهم» است. Go مجبورت می‌کند
> هر مقدار را مدیریت کنی، اما `_` می‌گذارد مؤدبانه یکی را دور بیندازی.

فایل [`examples/session04/loops/loops.go`](../examples/session04/loops/loops.go) را اجرا کن.

## ۲. توابع — بلوک‌های منطق قابل‌استفاده‌ی مجدد (۲۰ دقیقه)

یک **تابع** کد را بسته‌بندی می‌کند تا با نام صدایش بزنی، به آن ورودی (**پارامتر**)
بدهی و خروجی (**مقدار بازگشتی**) بگیری.

### ساختار

</div>

```go
func add(a int, b int) int {
    return a + b
}
```

<div dir="rtl">

| بخش | معنی |
|-----|------|
| `func` | کلیدواژه‌ای که تابع را شروع می‌کند |
| `add` | نام تابع |
| `(a int, b int)` | پارامترها با نوعشان |
| `int` (بعد از `)`) | **نوع بازگشتی** |
| `return a + b` | مقداری به فراخواننده برمی‌گرداند |

این‌طور صدایش بزن:

</div>

```go
sum := add(3, 4)   // sum == 7
```

<div dir="rtl">

### کوتاه‌نویسی پارامترهای هم‌نوع

اگر پارامترهای پشت‌سرهم یک نوع دارند، نوع را یک بار بنویس:

</div>

```go
func add(a, b int) int {   // مثل (a int, b int)
    return a + b
}
```

<div dir="rtl">

### توابع بدون بازگشت

اگر تابع چیزی برنمی‌گرداند، نوع بازگشتی را حذف کن:

</div>

```go
func greet(name string) {
    fmt.Println("Hello,", name)
}
```

<div dir="rtl">

### کجا می‌توانی تابع را صدا بزنی؟

هرجای همان پکیج — **ترتیب مهم نیست**. می‌توانی `main` را بالا و توابع کمکی را
پایین تعریف کنی؛ Go پیدایشان می‌کند.

فایل [`examples/session04/functions/functions.go`](../examples/session04/functions/functions.go) را اجرا کن.

## ۳. کنار هم: FizzBuzz درست (۱۰ دقیقه)

حالا همه‌چیز را داری تا سوال کلاسیک مصاحبه را با حلقه درست حل کنی. راه‌حل کامل و
اصطلاحی در [`examples/session04/fizzbuzz/fizzbuzz.go`](../examples/session04/fizzbuzz/fizzbuzz.go) است:

</div>

```go
func fizzbuzz(n int) {
    for i := 1; i <= n; i++ {
        switch {
        case i%15 == 0:
            fmt.Println("FizzBuzz")
        case i%3 == 0:
            fmt.Println("Fizz")
        case i%5 == 0:
            fmt.Println("Buzz")
        default:
            fmt.Println(i)
        }
    }
}
```

<div dir="rtl">

ببین چطور `for`، `switch`، `%` و یک تابع کنار هم می‌آیند. **چرا اول `%15`؟** چون
عددی که بر ۳ و ۵ بخش‌پذیر است بر ۱۵ هم بخش‌پذیر است — و `switch` بدون شرط در اولین
انطباق متوقف می‌شود، پس مشخص‌ترین حالت باید اول بیاید.

</div>

```bash
go run examples/session04/fizzbuzz/fizzbuzz.go
```

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۰۵ انجام بده!)

فایل `examples/session04/practice/practice.go` را بساز:

۱. **جمع ۱ تا ۱۰۰:** با حلقه‌ی `for` همه‌ی اعداد ۱ تا ۱۰۰ را جمع و مجموع را چاپ کن. (پاسخ معروف ۵۰۵۰ است.)
۲. **شمارش معکوس:** از ۱۰ تا ۱ (`i--`) حلقه بزن، هرکدام را چاپ کن، بعد «Liftoff!» چاپ کن.
۳. **تابع `isPrime`:** `func isPrime(n int) bool` بنویس که اول بودن `n` را برگرداند. روی ۷، ۱۰ و ۱۳ تست کن.
۴. **تابع `factorial`:** `func factorial(n int) int` با حلقه بنویس که `n!` را برگرداند (مثلاً `factorial(5)` → 120).
۵. **ترکیب:** `func sumEven(limit int) int` بنویس که جمع همه‌ی اعداد زوج از ۱ تا `limit` را با استفاده از `continue` برای رد کردن فردها برگرداند.
